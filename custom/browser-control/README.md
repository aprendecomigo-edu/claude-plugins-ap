# browser-control

Browser automation and test generation plugin for Claude Code, providing browser control via `playwright-cli` and automated test development via Playwright's test agents.

## Overview

This plugin equips Claude with two complementary capabilities:

1. **playwright-cli skill** — Interactive browser automation via Anthropic's CLI (navigation, form filling, screenshots, scraping, session management, network mocking)
2. **Playwright test agents** — Three chained agents (Planner → Generator → Healer) that automatically plan, generate, and repair Playwright E2E tests

## Features

### Browser Automation (playwright-cli)
- Navigation, clicking, typing, form filling, drag-and-drop, keyboard/mouse events
- Session management with isolated browser contexts (`-s` flag)
- Storage management: cookies, localStorage, sessionStorage, state save/load
- Network request mocking and interception
- Multi-tab support
- Tracing and video recording for debugging

### Test Agents (Playwright)
- **Planner agent** — Explores the app and produces structured markdown test plans
- **Generator agent** — Transforms plans into executable Playwright test files by running steps live
- **Healer agent** — Runs tests, debugs failures, patches code, and re-runs until passing
- MCP server integration (`npx playwright run-test-mcp-server`)

## Installation

Add to your Claude Code project:

```bash
claude plugin add browser-control
```

## Prerequisites

For browser automation:

```bash
npm install -g playwright-cli
```

For test agents:

```bash
npm install -D @playwright/test
```

## Usage

### Browser Automation

Claude will automatically apply the `playwright-cli` skill when you ask it to automate browser interactions, scrape pages, fill forms, take screenshots, or mock network requests.

**Example prompts:**
- "Open example.com and take a screenshot"
- "Log into app.example.com and save the auth state"
- "Mock the /api/users endpoint to return test data"

### Test Agents

Claude will dispatch the appropriate test agent when you ask it to plan, generate, or fix tests.

**Example prompts:**
- "Create a test plan for the login flow"
- "Generate Playwright tests from the spec in specs/auth.md"
- "Fix the failing Playwright tests"
- "Plan and generate E2E tests for the dashboard"

## Agents

### playwright-test-planner

Explores a web app via browser and produces a comprehensive markdown test plan saved to `specs/`.

### playwright-test-generator

Reads a test plan from `specs/`, executes each step live against the app, and writes Playwright test files.

### playwright-test-healer

Runs the test suite, identifies failures, debugs interactively, patches test code, and re-runs until passing.
